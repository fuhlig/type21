The default base font size in browsers is 16px which equals 1em (100%). To set the text to 14px (87.5%) the font-size would be 0.875em. In order to avoid calculations it became best practice to set the base font-size to 62.5% which equals 10px. As 1em is 100% of the base font size, 1em becomes 10px aswell which makes calculations easier.

Font-size: 1em;     // 10px
Font-size: 1.4em;   // 14px
Font-size: 3.2em;   // 32px

The drawback of this solution is the nesting which means the font-size element of the element is relative to the parent.

Body    font-size: 62.5%;   // 1em = 10px
H1  	font-size: 2em;     // 20px
P   	font-size: 1.4em;   // 14px
P li    font-size: 1em;     // not 10px! (1 * 10px = 10px)
                            // 14px (1 * 14 = 14px)

Compared to em the root em is not relative to the parent but the document (html).

This is considered as a best practice especially with responsive design in mind.

The rem unit is supported by all modern browsers and devices except for legacy browsers (IE < 8). To deliver the right font-size for these browsers aswell a fallback is needed. If the browser doesnâ€™t support rem it takes the pixel value.

html { font-size: 62.5%; } 
body { font-size: 14px; font-size: 1.4rem; } /* =14px */
h1   { font-size: 24px; font-size: 2.4rem; } /* =24px */
